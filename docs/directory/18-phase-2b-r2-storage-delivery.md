# Parish Directory Phase 2B -- R2 Storage and Delivery

## Bucket Decision

Directory photos use a separate private R2 bucket:

- bucket: `agapay-directory-media`
- binding: `DIRECTORY_MEDIA`

The existing `CAMPAIGN_ASSETS` bucket has a public `r2.dev` URL and is meant for public campaign assets, so it is not appropriate for private directory photos.

The existing tax exemption and giving-statement buckets establish the private-bucket pattern used here: no public `r2.dev` access, no client-facing object keys, and Worker-mediated delivery.

## Object Keys

Object keys are generated by the server:

```text
directory/{environment}/{parishId}/{ownerType}/{ownerId}/{assetId}/{variant}.{ext}
```

Object keys are not treated as authorization. Every delivery request resolves the authenticated directory context and checks owner authority before streaming a variant.

## Delivery

Routes stream media through the Worker:

```text
GET /api/directory/media/:assetId/variants/:variantType
```

Responses include:

- `X-Content-Type-Options: nosniff`
- private cache headers
- no raw R2 credentials
- no unrestricted public URL

No signed URLs are used in this phase, avoiding a new credential surface.

## Manual Cloudflare Setup

The bucket must exist before deploy:

```text
npx wrangler r2 bucket create agapay-directory-media --location wnam
```

Public `r2.dev` access should remain disabled.
